      - DateTime input
      - Label: "End Time"
      - Validation: Must be after start time
   
   3. "Check Availability" Button
      - Primary blue
      - Fetches slots with time filter
      - Smooth scroll to slots section
   
   Validation Rules:
   - Start time must be in future
   - End time must be after start time
   - Minimum duration: 1 hour
   - Real-time error messages

C. SLOTS DISPLAY SECTION (id="slots-section")
   
   Grid Layout:
   - 2 columns on mobile
   - 3 columns on tablet
   - 4 columns on desktop
   
   Slot Card (SlotCard component):
   
   Visual Design:
   - Card with shadow
   - Status-based color scheme:
     * Available: Green border, white background
     * Occupied: Red border, red-50 background
     * Inactive: Gray border, gray-100 background
   
   Content:
   - Slot code (large, bold, centered)
   - Slot type badge:
     * Car: Blue badge
     * Bike: Purple badge
     * EV: Green badge
   - Status indicator:
     * "Available" (green text)
     * "Occupied" (red text)
     * "Inactive" (gray text)
   - "Book Now" button (available slots only)
   
   Slot States:
   - is_available: true ‚Üí Can book
   - is_available: false ‚Üí Disabled, grayed out
   - is_active: false ‚Üí Not bookable

D. BOOKING MODAL (Popup)
   
   Triggered: When user clicks "Book Now"
   
   Modal Content:
   - Header: "Confirm Booking"
   - Slot information display:
     * Slot code
     * Slot type
     * Parking lot name
   - Pre-filled information:
     * Start time (from date picker)
     * End time (from date picker)
   - Vehicle Number Input:
     * Text field
     * Placeholder: "Enter vehicle number (e.g., ABC1234)"
     * Required field
     * Validation: Not empty, trimmed
   - Calculated Amount:
     * Duration calculation
     * Rate √ó Hours
     * Display: "‚Çπ{amount}"
   - Action Buttons:
     * "Confirm Booking" (primary)
     * "Cancel" (secondary)

E. BOOKING PROCESS
   
   Steps:
   1. User selects start/end time
   2. Clicks "Check Availability"
   3. System fetches available slots
   4. User clicks "Book Now" on desired slot
   5. Modal opens with booking form
   6. User enters vehicle number
   7. System validates:
      - Times are valid
      - Minimum 1 hour duration
      - Vehicle number provided
   8. API call creates booking
   9. On success:
      - Alert: "Booking successful!"
      - Navigate to dashboard
   10. On failure:
       - Display error message
       - Keep modal open for correction

F. ERROR HANDLING
   
   Validation Errors:
   - "Start time must be in the future"
   - "End time must be after start time"
   - "Minimum parking time is 1 hour"
   - "Vehicle number is required"
   
   API Errors:
   - "Slot not available for selected time"
   - "Booking conflict detected"
   - "Failed to create booking"
   - Network errors

G. REAL-TIME UPDATES
   
   Socket Events:
   - Joins room: `parking_lot_${lotId}`
   - Listens for: 'slot_updated'
   - On update:
     * Refetch slots
     * Update availability
     * Visual refresh (no page reload)
   - Leaves room on navigation away

H. LOADING STATES
   - Initial page load spinner
   - "Checking availability..." during fetch
   - "Processing booking..." during submission
   - Disabled buttons during operations

I. RESPONSIVE DESIGN
   - Mobile: Single column, large tap targets
   - Tablet: 2-3 columns
   - Desktop: 4 columns, optimal spacing
   - Modal: Centered, responsive width
